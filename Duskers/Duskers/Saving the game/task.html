Theory

Before you can start implementing the stage, it might be a good idea to go over some theory.

In order to check if a file exists in a given directory, we can use the os.path module. The syntax is as follows:

os.path.isfile(path)

It returns a boolean value. We can use a relative path here to refer to a file in the same directory as our main script.

For example, os.path.isfile("save_file.txt") looks for a file named save_file.txt in the same folder as your script. If such a file, exists it returns True. Make sure you implement it this way for the tests.

Description

In this stage, we will take care of something very important in games: saving the game to a file and loading it. The program will also record the date and time of the save. The data mechanism will be very simple: 1 line = 1 save slot. The line should contain all the necessary information to load the game again at the same point it was saved.

Objectives

  1. The save file should be named save_file.txt. Make sure the program checks that the file exists in the same directory as the program; if not, it should be created.
  2. We will use 3 save slots. If there is no data in a save slot, it should display empty. The save slots are numbered, and the player enters the number of the slot they want to load or save to. Make sure that the player is not able to load an empty slot! All 3 slots should be saved in a single file with all the necessary information to restore the game in the exact same place after loading. This implies saving the player's name, the current Titanium balance, and Robot count. It should also contain the current time and date of saving in the following format: YYYY-MM-DD hh:mm. For example: 2020-09-24 20:05.
  3. Display a small dialog box if saving/loading was successful.
  4. Make sure you also add the saving functionality to the save and exit option.
  5. You should now replace Play with New game and Load game options in the menu.
