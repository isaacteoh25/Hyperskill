Description

Our game's core idea is exploration and adventure, so let's add a gameplay feature of location exploring. Places should have random names, as well as an amount of titanium that the player receives after exploration. The player will be able to look for places which should show up one by one. You should also take command-line arguments for us to test your game, including an argument for the seed, which is simply a starting point for the pseudo-random number generator that you should implement. You can read more about it in the Python Random Seed article or in the official docs.

Objectives

  1. Your program should take 4 command-line arguments with the seed being the first so that we can test the program. Next is the minimum duration of animations, then the maximum duration of animations; in both of these, "0" means disabled. The 4th argument contains the names of possible locations separated by a slash /. Note that each word in a location should be separated by a comma ,. If there are no arguments, the seed should be default, as well as the animation duration and location names (ones that you make up the names for).
  2. When the player chooses "Explore", the program should choose 1-9 locations to explore and then display them one by one after the user continues searching using manual input. Once there are no more locations to explore, the player can still choose a location or go back to the HUB. The locations should be numbered.
  3. To choose a location to explore, the player should enter the location number. The amount of titanium for each location should be predetermined but hidden from a player. The values range between 1 and 100.
  4. The back command should be available at any time.
  5. Let the player see on their HUB how much titanium they've collected. Display that value under the robots.
  6. Display the results after each exploration in the log part of the HUB. For example:
Deploying robots
High street was explored successfully, with no damage taken.
Acquired 87 lumps of titanium

In this stage, the game should be endless.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
