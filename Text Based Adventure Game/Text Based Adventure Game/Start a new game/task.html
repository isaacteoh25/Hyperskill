Description

In this stage, you will implement the new game function and the game stats.

You need a game function that allows you to store all the necessary items and use them in your playthrough. The game stats will be saved once the player advances to another level.

You need to store important variables when creating a new game such as the character attributes, inventory, difficulty, the remaining lives, level, and the username.

To create a new game, ask for the game variables, and if a player makes a mistake by choosing to create a new game, give them an option to go back to the main menu before creating a new player.

The character attributes include the Name, Species, and the Gender.

The inventory includes things like Snack, Weapon, Tool.

Difficulty levels are Easy, Medium and Hard.

The number of lives is 5, 3 or 1.

You should get these game variables in the same order as in the examples.

The Level starts with 1 (the first level). The game is going to have 2 levels in total at the end.

For the save file path, use %saves/<username>.txt%. In this case, each user will have their own save file. Don't worry, you won't implement saving progress in this stage, this path will be used in later stages.

Objectives

In this stage, your program should:

  1. Create and store the game state with variable inputs such as the inventory, character attributes, difficulty, remaining lives, and the username for the save file path.
  2. Have the option to go back to the menu before creating a new game. Don't forget that it should handle all the letter cases.
  3. Before the new game starts, output the message with the game stats such as the character traits, inventory, and difficulty.

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
