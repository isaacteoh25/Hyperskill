Description

In this stage, you are going to implement the core game features and only the first level of the game.

You are not supposed to save the game yet, that will come later.

First, you need to read the contents of the story.txt, choices.txt, and outcomes.txt. You can use the +, * signs to easily split these files except the choices.txt. For that you can use the new line character: \n These signs separate different chapters of the game. Download these files and paste the directory where you are working on a project. Save these files in the story folder.

In the choices.txt, you will see some of the words are wrapped in curly brackets like {weapon}. This means that you need to output the player's choice of weapon in these sentences. Output the right object in place of the bracketed word.

Also, in the outcomes.txt, you will find the outcomes for the player choices. Some of them are wrapped in the parentheses: (life+1). They indicate some of the outcomes, depending on a player's choice. You need to take them out of the sentences when you plan to use them in your story. Be careful as they are not a part of the story.

Below is the table of the possible outcomes:

life +1 increases the number of lives
life -1 decreases the number of lives
inventory + 'item_name' adds a new item to inventory
inventory - 'item_name' removes an item from inventory
move moves to the next segment of the story
repeat repeat the choices that were made in this segment of the story
save indicates the end of a level, saves the game
game_won a player has beaten the game, exits the program

These outcomes can be combined together to make two actions at the same time. For example: (life+1 and move)

Keep in mind that some of the outcomes such as save, game_won is going to appear in later stages. So, don't worry about them now.

In addition, you will see an outcome like this: 

You tried the key on the lock and the door opened.(inventory-'key' and move option1)

You don't have a key to open the lock. (repeat option2)

This means that an action made by the player has two outcomes depending on the player's previous actions. In this instance, for the option1 to happen, the player should have a key in the inventory. However, if the player doesn't have a key, the option2 happens.

Once you have the story, you can move to the core game loop. The game loop should check for the number of lives and if there are no lives remaining, the game is over. Also, check for the level of the game. If the player reaches the second level, your game should show it. However, since you are only implementing the first level in this stage, just output the Level 2 message and end the program.

Don't forget to print the Goodbye! message every time your program ends

Once the core game loop is set up, it's time to implement the gameplay features. Introduce your story to a player, then give a player some options to continue the game. The options should have an index so that the player can type the number to choose the option. Limit the options to three, just to keep it simple and show the outcome.

If your character dies, the game should start from the beginning of the level, decrease the number of the remaining lives but preserve the content of the inventory.

You also need to implement the following commands: /i, /c, /h:

/i shows the content of the inventory
/c shows the character traits and the remaining lives
/h the help page; shows the commands available to a player

We need to implement the outcomes next. If a player dies during the game or loses something from the inventory, your game should indicate it. The outcome variables may differ:

increase_lives increases the number of lives by 1 and shows a message with the remaining lives
decrease_lives decreases the number of lives by 1 and shows a message with the remaining lives.
add_item adds a new item to the inventory
remove_item removes an item from the inventory

The player should be able to exit the game by typing /q anytime. Don't forget to ask for the confirmation. Don't forget to show a message when an unknown input occurs.

You can always write your own story for this project. However, make sure that you use the same character traits and inventory items in your story. Also, make sure that in your story, the player always has the weapon, the tool and the snack.

Objectives

In this stage, your program should:

  1. Read the story folder content.
  2. Create a core game loop.
  3. Take inputs from the player for different actions.
  4. Increase the number of lives.
  5. Decrease the number of lives.
  6. Show the player inventory.
  7. Add an item to the inventory.
  8. Remove the item from the inventory.
  9. Show the character traits, including the remaining lives.
  10. Show the help page.
