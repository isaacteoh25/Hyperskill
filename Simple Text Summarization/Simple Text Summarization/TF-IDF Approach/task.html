Theory

In this stage, you will try out the TF-IDF approach. It is a modified word probability approach that is based on the two assumptions:

  1. The greater the word frequency (w) in the document, the more important the word is for this document.
  2. The less documents with the word w, the more important that word w is for the whole set of documents.

TF stands for "term frequency" and IDF is for "inverted document frequency". TF-IDF is a product of TF and IDF:

\(TF\text{-}IDF(t,d)=TF(t,d)*log(N/DF(t))\)

  • \(TF(t,d)\) is the number of occurrences of the word w in the document d;
  • \(DF(t)\) is the number of documents containing the word w;
  • N is the total number of documents.

Note that for this project, we consider one sentence as a whole document.

Check out machinelearningmastery.com for a comprehensive explanation and examples of using sklearn's TfIidfVectorizer.

Description

TF-IDF reflects the importance of a word for a specific document in a collection of documents. This measure was widely used by search engines to score and rank documents according to the user's query. It was also used for text classification and text summarization, two of the fundamental NLP-tasks. This method is not computationally expensive and yields great results for text summarization tasks.

In this stage, you will get familiar with some theory behind this method and work with one of the most popular machine learning libraries for Python sklearn.

Objectives

Steps 1-6 are the same as in the second stage. Your updated program should:

  1. Read the XML-file with news texts.
  2. Extract the headers and the news texts.
  3. Tokenize each text into separate sentences.
  4. Tokenize sentences into words.
  5. Get rid of punctuation and stopwords provided by NLTK.
  6. Lemmatize each word.
  7. Count the TF-IDF for each word and the mean TF-IDF for each sentence.
  8. Pick sqrt(N) best scoring sentences.
  9. Print the header and the selected sentences in the order in which they appear in the text.

Note that for this project, you should use a round function to transform sqrt(N) into an integer. The summarized news texts should consist of unprocessed sentences.

Tokenization into words (or tokens, to be more precise) is an extremely important step for NLP. There are lots of ways to do it - from regular expressions to machine learning. For this project you have to use word_tokenize() function from nltk. It returns list of words, punctuation and divides words like cat's into 2 parts - cat and 's (the second token shouldn't be removed). For the same reason we recommend to use WordNetLemmatizer(), but you can also try other lemmatizers.

Your program does not get any input in this stage: you should just read the news from the news.xml file. Do not make any changes to this file. The number of news pieces in one file may vary. You may find examples of its structure below.

All sentences start on a new line. If there are several sentences on one line, consider them all as one sentence.

Once you got the summaries for each piece of news, print them like you did in the previous stages.
